JSTL Bundle PLugin — Package to provide easy localization for a web application.

Stripes has some localization, but only for error messages and field names. If you want more, you can use the ubiquitous JSTL formatting tags. You can even configure it in your web.xml: 

    <config-param>
        <param-name>javax.servlet.jsp.jstl.fmt.LocalizationContext</param-name>
        <param-value>StripesResources</param-value>
    </config-param>

Note however, that this approach only works for simple ResourceBundles, i.e. with .properties files. The JSTL bundle plugin provides a means to go beyond this. Whenever you use the JSTL tags for localization and you also implement your own {@link net.sourceforge.stripes.localization.LocalizationBundleFactory}, you'll benefit from using it.

Using this plugin

The plugin has two modes of operation: reuse the JSTL default, or set the JSTL default.

To reuse the JSTL default, configure Stripes to use org.stripesstuff.plugin.jstl.JstlLocalizationBundleFactory as LocalizationBundleFactory. This ensures that Stripes uses your context parameter to load the ResourceBundle for error messages, field names and the default bundle in JSP's.

You can also set the JSTL default using the plugin. To do that, configure Stripes to use the {@link JstlBundleInterceptor}, either by explicit configuration, or by including this package as an extension package. Configuration notes can be found in the javadoc of {@link JstlBundleInterceptor}.

If you configure both approaches, the LocalizationBundleFactory approach takes precedence (the interceptor disable—ï itself).
